# jung_gpredict

`jung_gpredict` is a Windows desktop satellite pass prediction app inspired by gpredict. It accepts the 3-line TLE output generated by `TLE_Download` and shows upcoming AOS/LOS, azimuth, elevation, range, and range-rate information in a table.

## Current MVP

- WPF desktop app using C# and .NET.
- Drag and drop a `TLE_*.txt` file from `TLE_Download`.
- Select a Korean ground station from the list.
- Select which loaded satellites should be included in prediction.
- Show configurable visible passes for every selected satellite.
- Configure prediction days, maximum passes per satellite, and minimum elevation.
- Show gpredict-style multi-pass fields: AOS, TCA, LOS, duration, maximum elevation, AOS/max/LOS azimuth, range, and range rate.
- Show selected-pass summary details.
- Toggle table columns from the settings panel.

## gpredict Reference

The first design pass references `https://github.com/csete/gpredict`, especially its module-oriented view composition, satellite list, event list, multi-pass columns, and single-pass detail concepts. The app remains a .NET/WPF implementation rather than a gpredict source port.

## TLE Input Contract

The app expects repeated 3-line blocks:

```text
SATELLITE_NAME
1 ...
2 ...
```

This matches `D:\workspace\Codex_Workspace\TLE_Download\samples\TLE_260521_123037.txt`.

## Build

Install the .NET SDK, then run:

```powershell
dotnet restore .\jung_gpredict.sln
dotnet build .\jung_gpredict.sln
dotnet run --project .\src\Jung.Gpredict\Jung.Gpredict.csproj
```

This environment did not have `dotnet` available in PATH when the project was created, so the first local build should be done after installing or exposing the .NET SDK.
